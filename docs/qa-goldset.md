# 골드셋 질의 · QA 체크리스트 (WBS 1.7)

`PRD.md` §2·§8, `WBS.md` 1.7/4.1 기준의 실사용 QA 시나리오 고정본이다.
목표는 릴리스 전/후 회귀에서 "응답 문장 + 결과 카드" 품질을 같은 기준으로 검증하는 것이다.

## 1) 공통 합격 기준 (모든 케이스)

- [ ] 응답이 한국어이며 질문 의도를 벗어나지 않는다.
- [ ] 적어도 1개 이상의 tool 결과 기반 카드가 노출된다 (검색 후보 포함).
- [ ] 카드의 핵심 필드가 비정상적으로 깨지지 않는다 (`undefined`, `[object Object]` 금지).
- [ ] 재료 조건이 있는 서식지는 가능한 한 한글 재료명(`... x N`)으로 표시된다.
- [ ] 출처 필드는 값이 있으면 요약 문자열, 없으면 `정보 없음`으로 일관 표시된다.
- [ ] 오류 발생 시 앱이 멈추지 않고 사용자에게 에러 메시지를 노출한다.

## 2) 케이스별 합격 기준

- **포켓몬→서식지**: 서식지명 + (가능하면) 지역명이 목록으로 나온다.
- **서식지→포켓몬**: 포켓몬 목록이 1개 이상 나오거나, 없으면 명시적으로 "없음"을 안내한다.
- **서식지 상세**: 이벤트 여부/재료/지역 중 2개 이상 필드가 보인다.
- **조건 검색**: 질문 조건이 카드 칩(재료/날씨/시간/바이옴/지역)에 반영된다.
- **모호 질의**: `search_entity` 기반 후보 카드 또는 disambiguation 문구가 나온다.

## 6) 실패 분류 규칙 (간단)

- **D-Data**: 데이터 누락/불일치 (`지역 없음`, `출처 정보 없음` 등)
- **R-Retrieval**: tool 선택 실패, 엉뚱한 함수 호출
- **U-UI**: 카드 렌더링/문구/레이아웃 문제
- **M-Model**: 답변 문장 품질(환각, 근거 누락, 장황함)
- **S-System**: 연결 오류, 타임아웃, 5xx

이 분류를 메모에 같이 붙이면 4.1(자동/반자동 평가) 단계로 확장하기 쉽다.
